# Homepage Dashboard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a static HTML homepage dashboard that links to all Kubernetes services in the cluster.

**Architecture:** Static HTML/CSS served by Nginx Alpine container, deployed as a single-replica Deployment in a `homepage` namespace with a LoadBalancer service on port 30080.

**Tech Stack:** HTML5, CSS3, Nginx Alpine, Kubernetes

## Global Constraints

- Pure HTML/CSS, no JavaScript
- Dark theme matching supermarket frontend (`#0f172a` background)
- Image size < 50MB (nginx:alpine)
- imagePullPolicy: IfNotPresent for k3s local images
- Service type: LoadBalancer

---

## File Structure

```
k8s-projects/homepage/
├── Dockerfile          # Nginx Alpine container build
├── nginx.conf          # Nginx configuration

└── k8s/

    ├── deployment.yaml # Nginx deployment (1 replica)
    └── service.yaml    # LoadBalancer service
```

    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    gzip on;
    gzip_types text/html text/css application/javascript;

### Task 3: Create Kubernetes manifests

**Files:**
- Create: `k8s-projects/homepage/k8s/namespace.yaml`
- Create: `k8s-projects/homepage/k8s/deployment.yaml`
- Create: `k8s-projects/homepage/k8s/service.yaml`

**Interfaces:**
- Consumes: Docker image `homepage:latest`
- Produces: Kubernetes resources in `homepage` namespace

- [ ] **Step 1: Create namespace.yaml**

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: homepage
  labels:
    app: homepage
```

- [ ] **Step 2: Create deployment.yaml**

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: homepage
  namespace: homepage
  labels:
    app: homepage
spec:
  replicas: 1
  selector:
    matchLabels:
      app: homepage
  template:
    metadata:
      labels:
        app: homepage
    spec:
      containers:
        - name: homepage
          image: docker.io/library/homepage:latest
          imagePullPolicy: IfNotPresent
          ports:
            - containerPort: 80
              name: http
          resources:
            requests:
              memory: "16Mi"
              cpu: "10m"
            limits:
              memory: "32Mi"
              cpu: "50m"
          readinessProbe:
            httpGet:
              path: /
              port: 80
            initialDelaySeconds: 5
            periodSeconds: 5
          livenessProbe:
            httpGet:
              path: /
              port: 80
            initialDelaySeconds: 10
            periodSeconds: 10
```

- [ ] **Step 3: Create service.yaml**

```yaml
apiVersion: v1
kind: Service
metadata:
  name: homepage
  namespace: homepage
  labels:
    app: homepage
spec:
  type: LoadBalancer
  ports:
    - port: 80
      targetPort: 80
      protocol: TCP
      name: http
  selector:
    app: homepage
```

Run: `kubectl apply -f /home/roman/k8s-projects/homepage/k8s/ --dry-run=client`
Expected: No errors
